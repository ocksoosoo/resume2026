# [개인] 프론트엔드 개발자 자기소개 페이지 제작

## 1. 개발 기간

- 2026.05.29 ~ 2026.06.05 (약 1주일)

## 2. 프로젝트 개요

- **디자인 콘셉트**: 평소 선호하는 미니멀하고 심플한 아티클 레이아웃 구조 위에 포인트 색상 **라임 옐로우**를 사용
- **도전 과제**: 거친 텍스처와 볼드한 레이아웃이 특징인 **브루탈리즘(Brutalism) UI/UX** 스타일을 연습하고 다소 미숙했던 **CSS Grid** 시스템을 마스터하기 위해 시안 구상 후 구현

## 3. 개발 프로세스

- **레이아웃 설계**: 구현 전 최소 너비(min-width)와 각 섹션별 높낮이를 미리 고려하여 전체적인 틀을 잡는 것부터 시작
- **HTML 마크업**: 시맨틱 태그를 활용하여 마크업 진행
- **CSS 스타일링**: 굵은 선과 그림자, 라임 옐로우 색상, 장식 요소를 중심으로 스타일링 진행
- **1차 리팩토링**: 중복되는 스타일 코드와 클래스명을 정리하고 그리드 구조를 최적화
- **미디어 쿼리(반응형)**: 브레이크 포인트에 맞춰 미디어 쿼리 설정
- **2차 리팩토링**: 반응형 코드 추가로 혼란스러워진 스타일 시트를 다시 정리
- **애니메이션 적용**: 인터랙티브한 요소와 호버 효과를 추가

## 4. 주요 구현 기능

- 히어로 섹션의 SNS 아이콘 호버 시 관련 주소가 보이며 바로가기 링크로 연결
- 사용자의 시선을 설계하는 인터랙티브 aster 장식 요소를 구현. 첫 번째 히어로 섹션의 큰 Aster를 클릭하면 자연스럽게 다음 섹션으로 화면이 이동. 추후 JS 사용하여 부드럽게 전환되는 효과를 추가할 예정
- 'transform'과 'hover' 효과를 적극 활용한 애니메이션. 외곽선이 있는 박스들에 통일감 있는 호버 액션을 부여
- 그라데이션 배경, 굵고 어두운 드롭 섀도우 효과를 매치하여 브루탈리즘 컨셉 구현 노력
- 마지막 Works 섹션은 'Grid'와 'margin' 값을 세밀하게 조절하여 박스들이 감각적으로 서로 겹치는 듯한 입체적인 레이아웃을 구현

## 5. 회고

- 각 섹션의 높이나 간격을 초반에 명확한 규칙 없이 잡고 시작하다 보니 디테일하게 뜯어봤을 때 규칙적이지 않은 부분이 생긴 점이 아쉬움. 다음 프로젝트부터는 개발 효율성과 코드의 일관성을 위해 시작 단계에서부터 더 정교한 디자인 시스템과 구상안을 만들고 작업할 계획
- 예상보다 모바일 반응형 설정이 까다로워서 시간을 오래 씀. 직접 디버깅하고 고쳐 나가다 보니 반응형 레이아웃을 설정하는 방법을 학습할 수 있었음
- 첫 프로젝트인 만큼 AI를 주로 쓰기보다 자력으로 해보고 싶었으나, 아직 프로그래밍 감각이 다져지지 않아 결국 AI를 조금씩 활용함
- CSS만으로 표현하기 어려운 다양한 모션 효과들은 다음 프로젝트에서 JS나 라이브러리 학습 후 구현해볼 계획
